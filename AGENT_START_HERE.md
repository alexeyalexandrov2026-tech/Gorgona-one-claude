## 🎯 Твоя задача

### 1️⃣ Прочитай план
Откройте файл `Plan.md` в корне репозитория. Это твой полный задачник:
- 22 последовательных шага
- Для каждого шага — что нужно делать и как проверить
- Таблица зависимостей (какие файлы зависят от каких)
- Риски и как их избежать

### 2️⃣ Прочитай текущий статус
Откройте `HANDOFF.md`. Там:
- Что уже сделано
- Что дальше делать (STEP 0)
- Как обновлять этот файл

### 3️⃣ Выполняй шаги по очереди
**Начни со STEP 0** в Plan.md и выполняй пошагово:
1. Каждый шаг имеет чек-лист верификации — пройди его полностью
2. После каждого шага — коммит
3. **Сразу же** делай `git push` (не жди конца дня)

### 4️⃣ Обновляй HANDOFF.md
После каждой группы коммитов (Commit 1, Commit 2 и т.д.):
- Отметь что-то как DONE
- Напиши результаты чек-листа
- Напиши, что дальше делать
- **Перед тем как остановиться** — обнови файл и запуши

## 🚨 Правила безопасности (НИКОГДА не нарушай!)

| Правило | Почему |
|---------|--------|
| **Никогда не трогай branch `main`** | Это production. Любой push туда может сломать live-сайт |
| **Никогда не коннектись к production БД** | Можешь удалить/испортить реальные данные |
| **Никогда не коммитьте реальные секреты** | Можешь утечь API-ключи и пароли |
| **Всегда делай backup перед reset** | Если что-то пошло не так — нужна точка восстановления |
| **Проверяй каждый чек-лист перед переходом дальше** | Пропущенный шаг = сломанный функционал на следующем |

```
2. Выполни все файлы/действия

4. Если всё ✓ — переходи к следующему STEP
5. Если что-то ✗ — останови, разбери, исправь, потом коммит
```

## 🛑 Если что-то пошло не так

1. **Сразу останови работу** — не пытайся "пролезть дальше"
2. **Прочитай чек-лист** — видишь что не работает?
3. **Посмотри в Plan.md Risk Registry** — может там описано как это исправить?
4. **Откати последний коммит** если нужно: `git reset --soft HEAD~1`
5. **Исправь проблему**
6. **Коммит и пуш заново**
